All Use-cases

admin

  • [UC2] Manage Personal Details:This use-case describes the process whereby a user views and updates their personal demographic details stored on the system.

merchant

  • [UC8] Maintain Product Catalogue:The merchant wishes to update their product catalogue. They may wish to add, delete or update items within the catalogue. Updates might include changes to descriptions, prices (such as special offers) or a change to the delivery options that can be supported.
  • [UC3] View Sales:Allows merchants to view a report on sales activity for products that have been purchased via the system. The merchant can choose different report types, view the report online, print it or save it locally in a different formats.

order

  • [UC5] Checkout:Describes the checkout process whereby the buyer confirms the items they wish to purchase, enters delivery, billing addresses as well as payment information. Optionally items can be giftwrapped for a small additional fee.
  • [UC6] View Products:Allows the user to enter some search criteria and browse the resulting list of matching products. Drill-down allows more details to be displayed for items of interest.
  • [UC4] Add to shopping cart:The buyer adds a selected item to the shopping cart. The item will be retained within the list of items in the cart such that it is available when the buyer has finished shopping and wishes to pay for their goods.
  • [UC7] Manage Shopping Cart:Allows the user to examine the shopping cart to see what products it contains. It also allows the quantity to be changed or items to be removed from the cart.
  • [UC9] Manage Orders:Allows the user to view current (open) and previous orders and track order and delivery progress.

security

  • [UC1] Logon:This use-case describes the sequence of events necessary to logon to the system
